### Root Cause Analysis — WHY the architecture gaps exist

| Cause | Effect | Evidence |
|---|---|---|
| **Fast iteration without refactoring gates** | All 3 entry-point files grew organically past their hard limits | No CI gate enforced line-count limits; CLAUDE.md is guidance, not policy |
| **No monorepo tool (Turborepo/pnpm workspaces)** | Each service evolved independently; module system diverged | BFF=`"type": "module"` (ESM), telegram-bridge=CommonJS |
| **CLAUDE.md written as a design doc, not enforced** | Architecture contract has zero automated enforcement at PR level | No `enforce_file_limits.py` in CI; no pre-commit hook for line counts |
| **Consensus feature predates the feature-directory rule** | App.jsx absorbed it before Rule #4 existed | `src/features/consensus/` never created; consensus logic is 100% inline in App.jsx |
| **`main.py` accumulated every integration concern** | 3,617 lines where 600 was the ceiling; each addition was "just one more import" | Kafka, Triton, vLLM, drift monitoring, Prometheus, lifespan, all in one file |
| **`server.mjs` absorbed all BFF concerns** | 1,188 lines; gRPC analysis-server and Keycloak added in rushed PRs | `analysis-server.mjs` extracted but `server.mjs` not refactored accordingly |
| **`App.jsx` is the app shell AND feature container** | 749 lines; routing, Firebase init, Zustand stores, and consensus display all in one file | No page-level components exist; `pages/CollectiveConsciousness.jsx` never extracted |
| **Telegram bridge was a quick prototype** | CommonJS throughout, `userRegistry.js` inline, never retrofitted | `telegram-bridge/index.js` is 29,402 bytes with no extracted utilities |
