                        🎯 USAGE EXAMPLES
═══════════════════════════════════════════════════════════════════════════════

SIGNUP FLOW:
   1. User enters email (honeypot field is hidden)
   2. If honeypot filled → Silent rejection (bot detected)
   3. MIME verification on identity upload
   4. OTP sent to email
   5. Account created on verification

LOGIN FLOW:
   1. Check if IP is banned
   2. Verify email/password or Google OAuth
   3. Load user profile & permissions
   4. Session persists across refreshes

ADMIN FLOW:
   1. Enter master password + OTP
   2. Admin panel opens with click tracking active
   3. Clicking buttons recorded for activity audit
   4. >5 clicks/second → Panel locks, OTP required
   5. All actions logged to Firestore

AI DELIBERATION:
   1. User asks trading question
   2. Rate limit checked (1 per 5s)
   3. Gemini API called with security checks
   4. If Gemini fails (503) → Fallback to Mistral
   5. If Mistral fails → Fallback to Groq
   6. Response shown with metadata

FILE UPLOAD:
   1. User selects identity document
   2. MIME type verified (magic bytes)
   3. Polyglot/executable detection
   4. File compressed if JPEG
   5. Uploaded to Firebase Storage
   6. URL stored in database
